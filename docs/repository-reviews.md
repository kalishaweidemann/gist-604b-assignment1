## Repository 1

**Repository Name:**  OSMnx
**Link:**  https://github.com/gboeing/osmnx

### Purpose
**What does this project do?**
According to the README file, the project contains a Python package to "easily download, model, analyze, and visualize street networks and other geospatial features from OpenStreetMap."  

### Repository Structure
**What folders or files stand out?**
The repository contains many of the standard components (README, LICENSE, CONTRIBUTING, .gitignore), as well as a folder containing dozens of Python scripts.

### Documentation
**How helpful is the README?**
It's minimal but cleary written, with links to more detailed getting started guides, examples, and support.

### Activity
**Is the repository actively maintained?**
The most recent update was last month, but there have been more recent issues reported. 

### Key Observation
**One thing this project does well.**
Looks like this project would automate or scale routing models with minimal efforting / coding.

## Repository 2

**Repository Name:**  Geemap 
**Link:**  https://github.com/gee-community/geemap

### Purpose
**What does this project do?**
 According to the README file, Geemap is a Python package that "enables users to analyze and visualize Earth Engine datasets interactively within a Jupyter-based environment."

### Repository Structure
**What folders or files stand out?**
This project is much more extensive than the first repository I looked at. I'm glad to see folders for examples and tutorials, as it seems like it would take some time and effort to figure out how to use the package.

### Documentation
**How helpful is the README?**
It has a lot of great information and links, although it does grant a lot of real estate to marketing their *Earth Engine & Geemap* book. 

### Activity
**Is the repository actively maintained?**
 Yes, it has activity as recently as yesterday.

### Key Observation
**One thing this project does well.**
This project / package enables users who are more familiar with Python than JavaScript the ability to easily work with Google Earth Engine via a Jupyter-based environment.

## Reflection
**In 3–5 sentences, explain what you learned about how open source projects are organized.**
Looking across these repositories, most open source projects follow a similar structure with standard files like README, LICENSE, CONTRIBUTING, and folders containing the actual source code. The README usually serves as the entry point, explaining what the project does and linking to documentation, examples, or tutorials. Many projects also include example scripts or notebooks to help users understand how to apply the tools. Another thing I noticed is that activity on issues and recent commits can give a quick sense of whether a project is actively maintained.
