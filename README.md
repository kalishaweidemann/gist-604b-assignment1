# GitHub GIS Portfolio

## About
I work at the intersection of digital systems and spatial thinking, where user behavior meets landscapes and real-world context.

With a background in web strategy and a growing focus on geospatial analysis and remote sensing, I'm interested in developing workflows that emphasizes clarity, usability, and reproducibility across desktop GIS, Python, spatial databases, and web mapping.

## Projects
- [QGIS tutorials portfolio](https://github.com/kalishaweidemann/qgis-tutorials-portfolio)  
  Desktop GIS workflows including projections, attribute management, raster and vector analysis, spatial queries, and cartographic map production.

- [Python for GIS: pandas, GeoPandas, and Rasterio](https://github.com/kalishaweidemann/python-for-gis)  
  Structured geospatial analysis workflows in Python, integrating tabular, vector, and raster processing with reproducible code and testing.

- [PostGIS spatial database workflows](https://github.com/kalishaweidemann/postgis-spatial-database-workflows)  
  SQL-based spatial analysis in PostgreSQL/PostGIS, including geometry operations, spatial relationships, and database-driven GIS workflows.

- [OpenStreetMap & PostGIS spatial analysis](https://github.com/kalishaweidemann/osm-postgis-spatial-analysis)  
  End-to-end geospatial workflow combining Python, PostGIS, and OpenStreetMap data for reproducible spatial analysis and query-based insights.

- [WebGIS with Leaflet: Boston park features, bike network, and neighborhoods](https://github.com/kalishaweidemann/boston-leaflet-webgis-app)  
  Client-side web mapping application using Leaflet to visualize and interact with GeoJSON datasets in a browser-based environment.
